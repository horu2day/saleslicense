### 1. API 설명: 관련 클래스/메서드의 목적과 사용법

제공된 HmEG.md 문서 내용에는 `AddShapeCylinder` 또는 `AddShapeSphere`와 직접적으로 관련된 API 클래스나 메서드에 대한 정보가 없습니다. 3D 공간 내에 원통(Cylinder) 또는 구(Sphere)와 같은 특정 형태의 도형을 추가하는 기능을 명시적으로 보여주는 항목은 찾아볼 수 없습니다.

다만, `HmEG.Space` 네임스페이스에 `AutoMeshModelConverting` 클래스가 언급되어 있으나, 이는 특정 형태의 도형을 추가하는 기능과는 직접적인 관련이 없는 것으로 보입니다.

### 2. C# 코드 예제: 완전한 실행 가능한 코드

제공된 문서 내용에서 `AddShapeCylinder` 또는 `AddShapeSphere` API를 찾을 수 없으므로, 이를 활용한 C# 코드 예제를 제공할 수 없습니다.

### 3. 출처: 문서의 어느 부분(라인 번호)을 참조했는지

제공된 문서 내용 전체(라인 48 주변부터 생략된 부분까지)를 검색하였으나, `AddShapeCylinder` 또는 `AddShapeSphere`, 그리고 `Cylinder` 또는 `Sphere`와 같은 키워드를 통해 해당 API에 대한 직접적인 언급은 찾을 수 없었습니다.

`HmEG.Space` 네임스페이스는 라인 90에서 `AutoMeshModelConverting` 클래스와 함께 언급됩니다.

### 4. 주의사항: 사용 시 고려해야 할 사항

*   **문서의 한계:** 제공된 HmEG.md 문서 내용은 HmEG 라이브러리의 전체 문서가 아닐 수 있으며, 특정 클래스와 메서드의 목록만을 보여주고 있습니다. 따라서 `AddShapeCylinder` 또는 `AddShapeSphere`와 같은 기능이 HmEG 라이브러리에 존재하더라도, 이 문서에는 포함되어 있지 않을 수 있습니다.
*   **기능 확인 필요:** 만약 이러한 도형 추가 기능이 필요하다면, HmEG 라이브러리의 다른 문서나 공식 API 레퍼런스를 참조하여 `HmEG.Space` 또는 `HmEG.Model`, `HmEG.Core` 네임스페이스 내에서 도형 생성 또는 추가와 관련된 클래스/메서드를 직접 찾아봐야 합니다. 일반적으로는 `MeshBuilder`, `GeometryHelper`, `SceneNode` 등에 이러한 기능이 포함될 수 있습니다.
*   **추측 금지:** 제공된 문서만을 기반으로 특정 API의 존재 여부나 사용법을 추측하는 것은 올바르지 않습니다.
